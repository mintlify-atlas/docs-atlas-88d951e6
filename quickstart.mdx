## Overview

This guide will help you integrate Autumn into your application. You'll learn how to:

- Install the Autumn SDK
- Configure authentication
- Create your first product
- Implement attach, check, and track functions

<Note>
  **Using cloud?** Sign up at [app.useautumn.com](https://app.useautumn.com) to get started instantly.
  
  **Self-hosting?** Follow our [self-hosting guide](/self-hosting) first, then return here.
</Note>

## Installation

<Steps>
  <Step title="Install the SDK">
    Add the Autumn SDK to your project using your preferred package manager.

    <CodeGroup>
      ```bash npm
      npm install autumn-js
      ```

      ```bash yarn
      yarn add autumn-js
      ```

      ```bash pnpm
      pnpm add autumn-js
      ```

      ```bash bun
      bun add autumn-js
      ```
    </CodeGroup>
  </Step>

  <Step title="Get your API key">
    Navigate to the Autumn dashboard and generate a secret key. Add it to your environment variables.

    ```bash .env
    AUTUMN_SECRET_KEY=am_sk_your_secret_key_here
    ```

    <Warning>
      Keep your secret key secure! Never commit it to version control or expose it in client-side code.
    </Warning>
  </Step>

  <Step title="Set up backend authentication">
    Mount the Autumn handler to your backend to enable the React hooks. This creates routes at `/api/autumn/*`.

    <Tabs>
      <Tab title="Next.js">
        Create a catch-all route at `app/api/autumn/[...all]/route.ts`:

        ```typescript app/api/autumn/[...all]/route.ts
        import { autumnHandler } from "autumn-js/next";
        import { auth } from "@/lib/auth";
        import { headers } from "next/headers";

        export const { GET, POST } = autumnHandler({
          identify: async () => {
            const session = await auth.api.getSession({
              headers: await headers(),
            });

            return {
              customerId: session?.user.id,
              customerData: {
                name: session?.user.name,
                email: session?.user.email,
              },
            };
          },
        });
        ```
      </Tab>

      <Tab title="Express">
        ```javascript server.js
        import express from "express";
        import { autumnHandler } from "autumn-js/backend";

        const app = express();
        app.use(express.json());

        app.use("/api/autumn/*", async (req, res) => {
          // Your authentication logic here
          const customerId = "user_id";

          const { statusCode, response } = await autumnHandler({
            customerId,
            customerData: { name: "", email: "" },
            request: {
              url: req.url,
              method: req.method,
              body: req.body,
            },
          });

          res.status(statusCode).json(response);
        });
        ```
      </Tab>

      <Tab title="Hono">
        ```typescript app.ts
        import { Hono } from "hono";
        import { autumnHandler } from "autumn-js/backend";

        const app = new Hono();

        app.use("/api/autumn/*", async (c) => {
          // Your authentication logic here
          const customerId = "user_id";

          let body = null;
          if (c.req.method !== "GET") {
            body = await c.req.json();
          }

          const { statusCode, response } = await autumnHandler({
            customerId,
            customerData: { name: "", email: "" },
            request: {
              url: c.req.url,
              method: c.req.method,
              body: body,
            },
          });

          return c.json(response, statusCode);
        });
        ```
      </Tab>
    </Tabs>

    <Tip>
      The `customerId` should be a unique identifier for your user or organization. This links Autumn's billing data to your users.
    </Tip>
  </Step>

  <Step title="Add the React provider">
    Wrap your application with the `AutumnProvider` to enable React hooks throughout your app.

    ```tsx app/layout.tsx
    import { AutumnProvider } from "autumn-js/react";

    export default function RootLayout({
      children,
    }: {
      children: React.ReactNode;
    }) {
      return (
        <html lang="en">
          <body>
            <AutumnProvider
              backendUrl="http://localhost:8000"
            >
              {children}
            </AutumnProvider>
          </body>
        </html>
      );
    }
    ```

    <Note>
      If your backend URL is different from your client URL, pass it as the `backendUrl` prop.
    </Note>
  </Step>
</Steps>

## Create your first product

<Steps>
  <Step title="Access the dashboard">
    Navigate to the Products page in your Autumn dashboard.
  </Step>

  <Step title="Create a product">
    Click **Create Product** and configure:
    - **Name**: e.g., "Pro Plan"
    - **ID**: e.g., `pro_plan` (used in your code)
    - **Price**: Set your pricing
    - **Features**: Add features like "api_calls" or "ai_tokens"
  </Step>

  <Step title="Configure features">
    For each feature, set:
    - **Feature ID**: Used in `check()` and `track()` calls
    - **Limit**: Maximum usage allowed
    - **Reset period**: How often limits reset (daily, monthly, etc.)
  </Step>
</Steps>

## Implement core functions

Now that your product is configured, implement the three core Autumn functions:

### 1. Attach a product

Allow customers to subscribe or upgrade to a plan:

```tsx components/UpgradeButton.tsx
import { useCustomer } from "autumn-js/react";

function UpgradeButton() {
  const { attach } = useCustomer();

  const handleUpgrade = async () => {
    await attach({
      productId: "pro_plan",
      successUrl: "http://localhost:3000/success",
    });
  };

  return (
    <button onClick={handleUpgrade}>
      Upgrade to Pro
    </button>
  );
}
```

<Tip>
  For new customers, `attach()` redirects to Stripe Checkout. For existing customers, it processes the change immediately.
</Tip>

### 2. Check feature access

Verify whether a customer can use a feature before allowing access:

```tsx components/FeatureButton.tsx
import { useCustomer } from "autumn-js/react";

function AIGenerateButton() {
  const { check } = useCustomer();

  const handleGenerate = async () => {
    const { data } = await check({
      featureId: "ai_tokens",
    });

    if (!data?.allowed) {
      alert("You've reached your AI generation limit!");
      return;
    }

    // Proceed with AI generation
    await generateAIContent();
  };

  return (
    <button onClick={handleGenerate}>
      Generate with AI
    </button>
  );
}
```

### 3. Track usage

Record usage events to enforce limits and calculate billing:

```tsx lib/ai.ts
import { useCustomer } from "autumn-js/react";

async function generateAIContent() {
  const { track } = useCustomer();

  // Generate content...
  const tokensUsed = 1500;

  // Record the usage
  await track({
    featureId: "ai_tokens",
    value: tokensUsed
  });

  return content;
}
```

<Note>
  Always call `track()` after the user has consumed the resource, not before. This ensures accurate billing.
</Note>

## Advanced usage

### Use the pre-built pricing table

Autumn includes a ready-to-use pricing table component:

```tsx pages/pricing.tsx
import { PricingTable } from "autumn-js/react";

export default function PricingPage() {
  return (
    <div className="w-full max-w-4xl mx-auto p-8">
      <PricingTable
        checkoutParams={{
          successUrl: "http://localhost:3000/welcome",
        }}
      />
    </div>
  );
}
```

### Get customer billing state

Access the customer's current billing state and product details:

```tsx components/BillingDashboard.tsx
import { usePricingTable } from "autumn-js/react";

function BillingDashboard() {
  const { products } = usePricingTable();

  return (
    <div>
      {products.map((product) => (
        <div key={product.id}>
          <h3>{product.name}</h3>
          <p>Status: {product.scenario}</p>
          {/* scenario can be: active, upgrade, downgrade, scheduled */}
        </div>
      ))}
    </div>
  );
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Authentication errors">
    If you're seeing authentication errors:
    - Verify your `AUTUMN_SECRET_KEY` is set correctly
    - Ensure the backend handler is properly mounted at `/api/autumn/*`
    - Check that `customerId` is being extracted correctly from your auth system
  </Accordion>

  <Accordion title="Attach not redirecting to Stripe">
    For first-time customers, `attach()` should redirect to Stripe Checkout. If it's not:
    - Verify your Stripe API keys are configured in the Autumn dashboard
    - Check that the `productId` matches a product in your dashboard
    - Ensure the customer doesn't already have an active subscription
  </Accordion>

  <Accordion title="Check always returns false">
    If `check()` is denying access unexpectedly:
    - Verify the `featureId` matches a feature in your product configuration
    - Check if the customer has an active subscription to a plan with that feature
    - Review the usage limits configured for that feature
  </Accordion>
</AccordionGroup>
